# Kensington 트랙볼 매핑 도구 (ktrackball) — 작업 정리

> Pop!_OS 24.04 (COSMIC, Wayland)용 Kensington 트랙볼 버튼 매퍼.
> **상태: 완료 · GitHub 공개됨** — https://github.com/eondcom/kensington-trackball-linux
> 최종 업데이트: 2026-05-31

---

## 1. 목표 (달성)

Kensington 트랙볼용 Linux 공식 드라이버(KensingtonWorks)가 없어서, 트랙볼 버튼을
브라우저 동작 등으로 매핑하는 도구를 직접 제작. 버튼 위치별 동작 지정 + 동시누름(코드)
+ 볼 스피드 조정까지 GUI로 설정 가능.

## 2. 환경

| 항목 | 값 |
|------|-----|
| OS / 데스크톱 / 세션 | Pop!_OS 24.04 LTS / COSMIC / **Wayland** |
| 트랙볼 | Kensington Expert Wireless (K72359WW), USB `047d:8018` |
| (보유) | Kensington Slimblade, USB `047d:2041` |

**핵심 제약:** Wayland라 X11 도구(`xinput`/`xdotool`) 불가 → 커널 레벨 `evdev`(읽기)
+ `uinput`(주입) 방식. compositor 무관하게 동작.

## 3. 설계 / 동작 방식

1. 트랙볼 포인터 디바이스를 **독점 grab**(`EVIOCGRAB`).
2. 가상 `uinput` 디바이스로 일반 커서/스크롤은 그대로 전달.
3. 물리 버튼은 **액션**으로 치환:
   - `passthrough`(일반 클릭) · `key`(키조합 주입) · `command`(프로그램 실행)
   - `scroll_hold`(누르고 볼 굴리면 스크롤) · `precision_hold`(누르는 동안 느리게)
4. **코드(chord):** 두 버튼 동시누름 → 단일 동작 억제하고 코드 동작 발동
   (`chord_window_ms` 동안 파트너 대기; key/command만 허용).
5. **볼 스피드:** `pointer_speed` 배율을 모든 커서 이동에 적용(마우스 감도처럼, 소수 누산).
6. 디바이스 매칭은 `device_match`(이름 부분일치) → USB 리시버·블루투스·두 트랙볼 모두 자동 인식.
7. systemd 서비스로 root 실행, 부팅 시 자동 시작, 무선 재연결 자동 재그랩.

## 4. 파일 구성 (`~/kensington-trackball/` = git 리포)

| 파일 | 설명 |
|------|------|
| `trackball_mapper.py` | 데몬 + CLI (`run`/`learn`/`learn-once`/`apply`/`list`/`check-config`) |
| `ktrackball_gui.py` | GTK3 GUI (6칸 위치 모델 + 볼 스피드 슬라이더) |
| `config.toml` | 디바이스 매칭 + 버튼/코드/스피드 설정 |
| `ktrackball.service` | systemd 자동시작 유닛 |
| `ktrackball.desktop` | 앱 메뉴 런처 |
| `install.sh` | 설치 스크립트 (sudo; 의존성·서비스·런처까지) |
| `README.md` / `LICENSE` / `.gitignore` | 문서 / MIT / 무시규칙(`.claude/` 제외) |
| `docs/screenshot.png` | GUI 스크린샷 |
| `작업정리.md` | 이 문서 |

### 설치 위치 (install.sh 후)
- 데몬·GUI: `/opt/ktrackball/` · 설정: `/etc/ktrackball/config.toml`
- 유닛: `/etc/systemd/system/ktrackball.service` · 런처: `/usr/share/applications/ktrackball.desktop`

## 5. 주요 명령

```bash
cd ~/kensington-trackball && sudo ./install.sh        # 설치

# 코드/GUI/설정 갱신 후 재배포
sudo cp ~/kensington-trackball/{trackball_mapper.py,ktrackball_gui.py} /opt/ktrackball/
sudo systemctl restart ktrackball

python3 /opt/ktrackball/ktrackball_gui.py             # GUI 실행 (또는 앱 메뉴)
python3 /opt/ktrackball/trackball_mapper.py list      # 입력장치 이름 확인
sudo python3 /opt/ktrackball/trackball_mapper.py learn --device /dev/input/by-id/usb-Kensington_Expert_Wireless_TB-event-mouse
journalctl -u ktrackball -f                           # 로그
```

## 6. 확정된 버튼 / config 형식

물리 버튼 4개(코너): `BTN_LEFT` `BTN_RIGHT` `BTN_MIDDLE` `BTN_SIDE` (**BTN_EXTRA 없음**).
스크롤링(휠)은 네이티브 동작. GUI 6칸: 좌상/우상/상단합/좌하/우하/하단합.

```toml
device_match = ["Expert Wireless", "Slimblade"]
chord_window_ms = 40
pointer_speed = 1.0

[positions]                       # GUI 메타데이터 (데몬은 무시)
top_left = "BTN_SIDE"
top_right = "BTN_MIDDLE"
bottom_left = "BTN_LEFT"
bottom_right = "BTN_RIGHT"

[buttons]
BTN_SIDE   = { type = "key", keys = ["KEY_LEFTALT", "KEY_LEFT"]  }   # 뒤로
BTN_MIDDLE = { type = "key", keys = ["KEY_LEFTALT", "KEY_RIGHT"] }   # 앞으로
BTN_LEFT   = { type = "passthrough" }
BTN_RIGHT  = { type = "passthrough" }

[[chords]]                        # 상단 두 버튼 동시 → 새 탭
buttons = ["BTN_SIDE", "BTN_MIDDLE"]
type = "key"
keys = ["KEY_LEFTCTRL", "KEY_T"]
```

## 7. GUI 구조 (`ktrackball_gui.py`)

- GTK3(PyGObject), **사용자 권한 실행**(Wayland 호환). 권한 필요한 2가지만 `pkexec`:
  - **버튼 감지** → `learn-once`: 데몬 중지 → 1회 입력 캡처 → 이름 출력 → 데몬 복구
  - **저장** → `apply <tmp>`: config 검증 → `/etc`(root:root 0644) 설치 → 데몬 재시작
- 6칸 위치 모델: 단일 4칸은 감지로 코드 확인, 동시(코드) 2칸은 자동 활성화.
  버튼 위치는 `[positions]`로 저장돼 다음 실행 때 복원.
- 동작 프리셋: 일반클릭/뒤로/앞으로/홈(Alt+Home)/Home/End/PageUp·Down/맨위·아래/
  새탭/탭닫기/새로고침/복사/붙여넣기/볼스크롤/정밀모드/사용자키조합/명령실행 + 볼 스피드 슬라이더.

## 8. 보안 (공개 리포 기준)

- 비밀정보·자격증명·텔레메트리 없음. PII 제거, `.claude/` 로컬설정 제외.
- 데몬 root 실행 = `/dev/input` 읽기(키 입력 감시 가능) + `/dev/uinput` 주입.
  모든 입력 리매퍼(input-remapper, keyd)와 동일한 본질적 권한 → 신뢰 코드만 실행.
- `/etc/ktrackball/config.toml` **root:root 0644 유지**가 핵심 안전장치
  (비root 쓰기 시 `command` 주입으로 권한 상승 가능 → install/`apply`가 소유권 강제).
- GUI는 root로 안 돌고 `pkexec`로 매번 인증. **무인증 polkit 규칙 추가 금지.**

## 10. 트러블슈팅 기록 (해결됨, 참고용)

**USB 연결 불량:** 작업 중 트랙볼이 인식 ~3초 후 끊기고 `Cannot enable / unable to
enumerate USB device` 반복 → **나노 리시버 접촉 불량**이 원인. **리시버를 다시 꽂으니 해결**.
핫플러그·소프트웨어 문제 아니었음(autosuspend도 무관했음).

> 재발 시 분기: ① 리시버 직결(허브/연장 케이블 배제) ② 다른 PC로 리시버 사망 여부 판별
> ③ 블루투스로 우회(코드가 BT 이름도 자동 인식) ④ autosuspend 의심 시 아래 udev 규칙.
>
> ```
> # /etc/udev/rules.d/50-kensington-noautosuspend.rules
> ACTION=="add", SUBSYSTEM=="usb", ATTR{idVendor}=="047d", ATTR{idProduct}=="8018", TEST=="power/control", ATTR{power/control}="on"
> # sudo udevadm control --reload && sudo udevadm trigger
> ```
